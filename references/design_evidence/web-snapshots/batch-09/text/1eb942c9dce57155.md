## What is Apache Cassandra<span class="image"><img src="_images/registered.svg" width="24" height="24" alt="Registered" /></span>?

### Apache Cassandra is an open source NoSQL distributed database trusted by thousands of companies for scalability and high availability without compromising performance. Linear scalability and proven fault-tolerance on commodity hardware or cloud infrastructure make it the perfect platform for mission-critical data.

<div class="openblock features-wrapper cf">

<div class="content">

<div class="openblock feature text-center ma-small my-large pa-small">

<div class="content">

<div class="paragraph">

<span class="image">![hybrid](_images/Icons/hybrid.png)</span>

</div>

### Hybrid

<div class="paragraph">

Masterless architecture and low latency means Cassandra will withstand an entire data center outage with no data loss—across public or private clouds and on-premises.

</div>

</div>

</div>

<div class="openblock feature text-center ma-small my-large pa-small">

<div class="content">

<div class="paragraph">

<span class="image">![fault](_images/Icons/fault.png)</span>

</div>

### Fault Tolerant

<div class="paragraph">

Cassandra’s support for replicating across multiple datacenters is best-in-class, providing lower latency for your users and the peace of mind of knowing that you can survive regional outages. Failed nodes can be replaced with no downtime.

</div>

</div>

</div>

<div class="openblock feature text-center ma-small my-large pa-small">

<div class="content">

<div class="paragraph">

<span class="image">![scalable](_images/Icons/scalable.png)</span>

</div>

### Focus on Quality

<div class="paragraph">

To ensure reliability and stability, Cassandra is tested on clusters as large as 1,000 nodes and with hundreds of real world use cases and schemas tested with replay, fuzz, property-based, fault-injection, and performance tests.

</div>

</div>

</div>

<div class="openblock feature text-center ma-small my-large pa-small">

<div class="content">

<div class="paragraph">

<span class="image">![performant](_images/Icons/performant.png)</span>

</div>

### Performant

<div class="paragraph">

Cassandra <a href="http://vldb.org/pvldb/vol5/p1724_tilmannrabl_vldb2012.pdf" target="_blank" rel="noopener">consistently outperforms</a> popular NoSQL alternatives in benchmarks and real applications, primarily because of fundamental architectural choices.

</div>

</div>

</div>

<div class="openblock feature text-center ma-small my-large pa-small">

<div class="content">

<div class="paragraph">

<span class="image">![control](_images/Icons/control.png)</span>

</div>

### You’re In Control

<div class="paragraph">

Choose between synchronous or asynchronous replication for each update. Highly available asynchronous operations are optimized with features like Hinted Handoff and Read Repair.

</div>

</div>

</div>

<div class="openblock feature text-center ma-small my-large pa-small">

<div class="content">

<div class="paragraph">

<span class="image">![cloud native](_images/Icons/cloud_native.png)</span>

</div>

### Security and Observability

<div class="paragraph">

The audit logging feature for operators tracks the DML, DDL, and DCL activity with minimal impact to normal workload performance, while the fqltool allows the capture and replay of production workloads for analysis.

</div>

</div>

</div>

<div class="openblock feature text-center ma-small my-large pa-small">

<div class="content">

<div class="paragraph">

<span class="image">![decentral](_images/Icons/decentral.png)</span>

</div>

### Distributed

<div class="paragraph">

Cassandra is suitable for applications that can’t afford to lose data, even when an entire data center goes down. There are no single points of failure. There are no network bottlenecks. Every node in the cluster is identical.

</div>

</div>

</div>

<div class="openblock feature text-center ma-small my-large pa-small">

<div class="content">

<div class="paragraph">

<span class="image">![scalable](_images/Icons/scalable.png)</span>

</div>

### Scalable

<div class="paragraph">

Read and write throughput both increase linearly as new machines are added, with no downtime or interruption to applications.

</div>

</div>

</div>

<div class="openblock feature text-center ma-small my-large pa-small">

<div class="content">

<div class="paragraph">

<span class="image">![elastic](_images/Icons/elastic.png)</span>

</div>

### Elastic

<div class="paragraph">

Cassandra streams data between nodes during scaling operations such as adding a new node or datacenter during peak traffic times. Zero Copy Streaming makes this up to 5x faster without vnodes for a more elastic architecture particularly in cloud and Kubernetes environments.

</div>

</div>

</div>

</div>

</div>

</div>

</div>
