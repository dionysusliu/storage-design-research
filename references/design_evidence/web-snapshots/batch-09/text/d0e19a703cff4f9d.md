<div class="columns">

<div class="column scenarios is-7">

TiKV provides both raw and ACID-compliant transactional key-value API, which is widely used in online serving services, such as the metadata storage system for object storage service, the storage system for recommendation systems, the <a href="https://www.featurestore.org/" target="_blank">online feature store</a>, etc.

<div class="scenario content">

TiKV is also widely used as the storage layer for database management systems, for example:

- <a href="https://github.com/pingcap/tidb" target="_blank">TiDB</a>: An open-source MySQL compatible NewSQL database that supports Hybrid Transactional and Analytical Processing (HTAP) workloads.
- <a href="https://github.com/zhihu/zetta" target="_blank">Zetta</a>: An open-source NoSQL database that supports Transaction and Cloud Spanner like API.
- <a href="https://github.com/yongman/tidis" target="_blank">Tidis</a>: a Distributed NoSQL database, providing a Redis protocol API (string, list, hash, set, sorted set), written in Go.
- <a href="https://github.com/distributedio/titan" target="_blank">Titan</a>: A distributed implementation of Redis compatible layer based on TiKV.
- <a href="https://github.com/juicedata/juicefs" target="_blank">JuiceFS</a>: An open-source POSIX file system that is based on TiKV and S3.

</div>

## Why TiKV

<div class="columns is-multiline">

<div class="column is-one-third">

<div class="card">

<div class="card-header">

Low and stable latency

<div class="card-header-icon">

<span class="icon has-text-primary">![header icon](img/homepage/low-latency.svg)</span>

</div>

</div>

<div class="card-content">

<div class="content">

RawKV’s average response time less than 1 ms (P99=10 ms).

</div>

</div>

</div>

</div>

<div class="column is-one-third">

<div class="card">

<div class="card-header">

High scalability

<div class="card-header-icon">

<span class="icon has-text-primary">![header icon](img/homepage/high-scalability.svg)</span>

</div>

</div>

<div class="card-content">

<div class="content">

With the [Placement Driver](/docs/5.1/reference/architecture/terminology/#placement-driver-pd) and carefully designed [Raft](https://raft.github.io/) groups, TiKV excels in horizontal scalability and can easily scale to 100+ terabytes of data. Scale-out your TiKV cluster to fit the data size growth without any impact on the application.

</div>

</div>

</div>

</div>

<div class="column is-one-third">

<div class="card">

<div class="card-header">

Easy to use

<div class="card-header-icon">

<span class="icon has-text-primary">![header icon](img/homepage/easy-to-use.svg)</span>

</div>

</div>

<div class="card-content">

<div class="content">

Run a single command to deploy a TiKV cluster with everything you need for production environments. Easily scale out or scale in the cluster with [TiUP](/docs/5.1/reference/tiup/) or TiKV operator.

</div>

</div>

</div>

</div>

<div class="column is-one-third">

<div class="card">

<div class="card-header">

Easy to maintain

<div class="card-header-icon">

<span class="icon has-text-primary">![header icon](img/homepage/easy-to-maintain.svg)</span>

</div>

</div>

<div class="card-content">

<div class="content">

TiKV is based on the design of [Google Spanner](https://research.google/pubs/pub39966/) and [HBase](https://hbase.apache.org/), but simpler to manage without dependencies on any distributed file system.

</div>

</div>

</div>

</div>

<div class="column is-one-third">

<div class="card">

<div class="card-header">

Consistent distributed transactions

<div class="card-header-icon">

<span class="icon has-text-primary">![header icon](img/homepage/consistent-distributed-transactions.svg)</span>

</div>

</div>

<div class="card-content">

<div class="content">

Similar to Google’s [Spanner](https://research.google/pubs/pub39966/), TiKV (TxnKV mode) supports externally consistent distributed transactions.

</div>

</div>

</div>

</div>

<div class="column is-one-third">

<div class="card">

<div class="card-header">

Adjustable consistency

<div class="card-header-icon">

<span class="icon has-text-primary">![header icon](img/homepage/adjustable-consistency.svg)</span>

</div>

</div>

<div class="card-content">

<div class="content">

In RawKV and TxnKV modes, you can customize the balance between consistency and performance.

</div>

</div>

</div>

</div>

</div>
